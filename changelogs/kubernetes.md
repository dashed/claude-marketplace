# Changelog - kubernetes

All notable changes to the kubernetes skill in this marketplace will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).

## [1.0.0] - 2026-06-11

### Added
- Initial addition to the marketplace — a skill for **Kubernetes**, scoped to the `kubectl` CLI + core resource/manifest authoring layer (cluster install/admin deferred to the sibling **k3s** skill; client libraries, cloud-provider specifics, and Helm out of scope). Authored against the Kubernetes source (dev tree ~1.37); CLI items empirically verified on the installed kubectl **v1.28.2** (client-side, no cluster).
- `SKILL.md`: declarative desired-state mental model (control loops, API groups, namespaces); the k3s cross-link; kubeconfig/contexts; six core workflows (deploy+expose, inspect a failing pod, apply/diff/rollout, exec/logs/port-forward, offline manifest generation with `--dry-run=client`, RBAC checks with `auth can-i`); quick-reference; Pod-status troubleshooting (CrashLoopBackOff/ImagePullBackOff/Pending/OOMKilled).
- `references/kubectl.md`: resource verbs, output formats (`-o jsonpath`/`custom-columns`/…), selectors, client vs **server-side apply** (+ conflicts, `--prune` caveat), diff, rollout, exec/logs/cp/port-forward, `kubectl debug` (ephemeral containers, GA 1.25), drain/cordon/taint, `auth can-i`/`whoami`, explain/api-resources, kustomize, `kubectl events` (standalone, 1.26), `--subresource`, scripting patterns. Every flag verified against kubectl 1.28.2 `--help`. Documents which `kubectl create` generators work fully offline — and that `run`/`expose`/`autoscale`/`create role --resource` contact the API server even under `--dry-run=client` (verified: they time out with no cluster).
- `references/workloads.md`: Pod spec essentials (env, resources/QoS, probes, lifecycle, securityContext, init containers, **native sidecars** via init-container `restartPolicy: Always` — GA 1.33, beta-on 1.29), Deployment/ReplicaSet/StatefulSet/DaemonSet, Job (podFailurePolicy GA 1.31, backoffLimitPerIndex/successPolicy GA 1.33), CronJob (timeZone GA 1.27), HPA `autoscaling/v2`, PDB, scheduling (affinity, topologySpreadConstraints, tolerations, priorityClassName). Field shapes verified against `staging/src/k8s.io/api/`.
- `references/cluster-resources.md`: Service types + headless, Ingress (+ Gateway API deferral), NetworkPolicy default-deny patterns, ConfigMap/Secret (types, immutable, update-propagation caveats), PV/PVC/StorageClass (ReadWriteOncePod GA 1.29, expansion GA 1.24, VolumeAttributesClass GA 1.34), Namespace/Quota/LimitRange, RBAC, ServiceAccount bound tokens (`expirationSeconds` min 600 verified against API source), DNS conventions, downward API, kubeconfig structure.
- `references/version-features.md`: 41 source-cited `feature → minimum Kubernetes version` rows (annotated by GA version, beta-on noted; e.g. server-side apply 1.22, ephemeral containers 1.25, sidecars 1.33, in-place pod resize 1.35, ValidatingAdmissionPolicy 1.30), plus explicitly-flagged not-yet-GA items. Sources: per-minor `CHANGELOG/CHANGELOG-1.x.md` + feature-gate stage metadata in `pkg/features/kube_features.go`, cross-checked with kubernetes.io and the installed kubectl.
- Inline `(k8s 1.X+)` annotations throughout; bedrock (GA ≤ ~1.20, Dec 2020) left unannotated ("unlisted = long-standing").

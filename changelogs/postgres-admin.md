# Changelog - postgres-admin

All notable changes to the postgres-admin skill in this marketplace will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).

## [1.0.0] - 2026-06-09

### Added
- Initial addition to the marketplace — a skill for **PostgreSQL server administration & operations** (DBA work on a running cluster), scoped to the server/ops surface and cross-linked to its sibling skills (`psql` client, `postgres-sql` dialect, `postgres-performance` query tuning/autovacuum mechanics, `postgres-extensions` contrib). Authored against the PostgreSQL source dev tree **19beta1** (stable = PG 18); every version annotation verified against the official postgresql.org release notes.
- `SKILL.md` (372 lines): cluster mental model; a related-skills disambiguation table; version-awareness/`SHOW server_version` check; seven operational sections (configuration & `ALTER SYSTEM`/reload-vs-restart, roles & privileges, authentication/`pg_hba.conf`, backup & recovery, WAL & replication, `pg_upgrade`, monitoring) with runnable examples; a quick-reference table; and an ops troubleshooting section.
- `references/config.md`: the three config layers, `pg_settings`/`pg_file_settings`, `ALTER SYSTEM`/`SET`/`ALTER ROLE`/`ALTER DATABASE`, reload-vs-restart by `context`, `postgresql.conf` syntax/includes, key parameters by area (memory/WAL/connections/autovacuum/logging), and units.
- `references/auth-roles.md`: roles vs users, attributes, `GRANT`/`REVOKE`, object privileges, `DEFAULT PRIVILEGES` (with the per-grantor gotcha), the pg15 `public`-schema hardening, role membership (pg16+ `WITH ADMIN/INHERIT/SET`), the full predefined-roles table with versions, and authentication (`pg_hba.conf` methods, SCRAM/`password_encryption`, `pg_ident.conf`, TLS).
- `references/backup-recovery.md`: logical vs physical decision table, `pg_dump`/`pg_dumpall`/`pg_restore` (formats, `-j`, `--section`, TOC `-l`/`-L`), `pg_basebackup`, incremental backups + `pg_combinebackup` (pg17+), manifests + `pg_verifybackup` (pg13+), continuous archiving + PITR (with the pg12 `recovery.conf` removal), and `pg_upgrade` (incl. `--swap` pg18+) plus minor-version upgrades.
- `references/replication.md`: WAL fundamentals, physical/streaming setup, replication slots (and the disk-fill hazard), synchronous-commit levels, hot standby, logical replication (publications with row filters/column lists pg15+, subscriptions, parallel apply pg16+), failover slot sync (pg17+), `pg_createsubscriber` (pg17+), and a physical-vs-logical chooser.
- `references/monitoring.md`: `pg_stat_activity` + cancel/terminate, `pg_stat_database`, `pg_stat_io` (pg16+), table/index/bloat signals, the progress-view table, replication monitoring, checkpoints/WAL/bgwriter (with the pg17 `pg_stat_checkpointer` split), locks (`pg_blocking_pids`), logging settings, stat resets, a triage playbook, and a pg19 monitoring round-up.
- `references/version-features.md`: ~75 source-cited `feature → minimum-PostgreSQL-version` rows (pg10 → pg19) grouped by area, with a "how to read"/bedrock section, per-version release-notes citations, and **(breaking)**/**(default)** markers — modeled on the duckdb skill's version-features reference.
- Inline `(pgNN+)` version annotations for features added in PostgreSQL 10+ where a release note sources them; bedrock (PostgreSQL 9.x or earlier, e.g. `ALTER SYSTEM` since 9.4) left unannotated. Notable precisely-verified facts: incremental backup + `pg_combinebackup` + failover slot sync (`sync_replication_slots`) = pg17; `pg_stat_io` = pg16; logical-replication row filters + column lists = pg15; SCRAM-SHA-256 = pg10 (default since pg14); predefined roles `pg_read_all_data`/`pg_write_all_data` = pg14, `pg_checkpoint` = pg15, `pg_create_subscription`/`pg_use_reserved_connections` = pg16, `pg_maintain` = pg17, `pg_signal_autovacuum_worker` = pg18; `pg_stat_checkpointer` = pg17 (breaking monitoring change); recovery.conf removal = pg12.

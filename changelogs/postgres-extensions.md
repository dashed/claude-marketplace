# Changelog - postgres-extensions

All notable changes to the postgres-extensions skill in this marketplace will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).

## [1.0.0] - 2026-06-09

### Added
- Initial addition to the marketplace — a skill for **PostgreSQL extension management and the bundled `contrib` ecosystem**: the `CREATE`/`ALTER`/`DROP EXTENSION` workflow plus a curated catalog of the modules shipped with PostgreSQL. Authored against the PostgreSQL source tree **19beta1** (stable = PG 18, PG 19 = beta); version facts verified against postgresql.org release notes and in-tree docs/`.control` files — never guessed.
- `SKILL.md` (~320 lines, under the 500-line budget): the extension mental model (a managed unit vs. loose SQL; contrib ships-but-dormant; **two independent version numbers**; preload-needing modules); cross-skill disambiguation vs. `psql`/`postgres-sql`/`postgres-performance`/`postgres-admin`; full management commands (install/discover/upgrade/relocate/remove); trusted extensions; the `shared_preload_libraries` restart set; a compact contrib catalog with runnable snippets grouped by purpose; external (non-contrib) PostGIS/pgvector/TimescaleDB note; troubleshooting; resources.
- `references/contrib-catalog.md`: the complete catalog of every bundled `contrib` module, grouped (text/fuzzy search, data types, indexing, FDW/external, crypto/UUID, query stats, storage forensics, table functions, SPI triggers, PL transforms, server hooks, client programs, deprecated), each with purpose + activation command + a tiny usage example + version notes. Clearly separates **`CREATE EXTENSION` modules** from **`LOAD`-only/preload modules** (`auto_explain`, `pg_overexplain`, `pg_plan_advice`, `passwordcheck`, `auth_delay`, …) and **client programs** (`oid2name`, `vacuumlo`).
- `references/management.md`: exhaustive management reference — `CREATE`/`ALTER`/`DROP EXTENSION` grammar, the `CASCADE` install-vs-drop direction distinction, the on-disk `.control`/SQL/upgrade-script file model, version-path upgrades, the system catalogs & views (`pg_extension`, `pg_available_extensions`, `pg_available_extension_versions`), schemas/relocatability/`search_path` gotchas, trusted-extension mechanics + full list, the `shared_preload_libraries` restart matrix, dump/restore & `pg_upgrade` behavior, and extension configuration tables.
- `references/version-features.md`: source-cited `extension/feature → minimum PostgreSQL version` map (PG 10 → PG 19), grouped by area, modeled on the duckdb skill's version-features file; includes the "how to read it" guide, a `default_version` ≠ PG-release explainer, a bedrock-modules table, and `SELECT version()` / `pg_extension` version-checking recipes.
- Inline `(pgNN+)` version annotations meaning the **PostgreSQL release** a module/feature first shipped in; modules present since the 9.x era are left **unannotated** ("bedrock"). Verified facts include: `gen_random_uuid()` graduated to **core in PG 13** (no extension needed; `pgcrypto`'s copy is now an obsolete wrapper); **trusted extensions** (non-superuser installs) added PG 13; `pg_surgery` PG 14; `pg_walinspect`/`basic_archive`/`basebackup_to_shell` PG 15; `daitch_mokotoff` (fuzzystrmatch) PG 16; `pg_buffercache_evict()` PG 17 and `adminpack` **removed** PG 17; `pg_overexplain`/`pg_logicalinspect` and core `uuidv7()`/`uuidv4()` PG 18; and the new PG 19 (beta) `pg_plan_advice`/`pg_stash_advice` modules.
- Emphasizes the recurring confusion that a module's `.control` `default_version` (e.g. `pg_stat_statements` 1.13) is the **extension's own** version and unrelated to the PostgreSQL server release.

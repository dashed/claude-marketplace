# Changelog - pytest

All notable changes to the pytest skill in this marketplace will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).

## [1.0.0] - 2026-06-15

### Added
- Initial addition to the marketplace — a skill for the **pytest testing framework** and the Python-specific idioms that set it apart from `unittest`. Scoped to pytest authoring/running/configuring; `unittest`, `tox`/`nox` runners, and coverage (`pytest-cov`) are explicitly disambiguated and deferred. Authored against the pytest source tree at **9.1.0 / 9.2.0.dev0** (`doc/en/`, `src/_pytest/`), with every version pin verified against the in-repo changelog (`doc/en/changelog.rst`).
- `SKILL.md` (321 lines): the mental-model shift vs `unittest` (plain `def test_*()` functions, bare `assert` rewritten for rich introspection, dependency-injected `@pytest.fixture`, table-driven `@pytest.mark.parametrize`, marks instead of class hierarchies — a from-inheritance-to-injection table); a disambiguation block (runs-but-isn't `unittest`; mocking via `monkeypatch`/`pytest-mock`; not `tox`/`nox`; not coverage); a when-to-use table; prerequisites + version note (latest stable 9.1.0, Python 3.10+, bedrock cutoff); seven runnable core workflows (write a test, fixtures by injection, parametrize, exceptions/warnings/approx, skip/xfail/markers, run & select, configure); a version-annotated quick-reference; troubleshooting; and an Advanced Usage section linking the nine references.
- `references/fixtures.md` (617 lines): defining/requesting & per-test caching, the `@pytest.fixture` decorator (scope/params/autouse/ids/name), all five scopes + dynamic/callable scope, `yield` teardown + `addfinalizer` + FILO finalizer order, autouse, the `request` object (full attribute/method table + marker introspection), parametrized fixtures, factory-as-fixture, `conftest.py` sharing & availability rules, overriding at every level, `usefixtures`, instantiation order, and `getfixturevalue`.
- `references/builtin-fixtures.md` (470 lines): `tmp_path`/`tmp_path_factory` (+ legacy `tmpdir`, retention/basetemp config), `monkeypatch` (all methods + `context()`/`undo()` + patch-where-used), `capsys`/`capsysbinary`/`capfd`/`capfdbinary`/`capteesys` (8.4+), `caplog` (+ propagation/`set_level`/`at_level`), `recwarn`, `request`, `cache`, `pytestconfig`, `record_property`/`record_testsuite_property`, `doctest_namespace`.
- `references/parametrize.md` (417 lines): `@pytest.mark.parametrize` (single/multiple args, force-tuple nuance), stacking → cartesian product, class/module `pytestmark`, `pytest.param(marks=/id=)`, custom ids (list/callable/per-case, `HIDDEN_PARAM` 8.4+, `strict_parametrization_ids` 9.0+), indirect parametrization via `request.param`, parametrizing fixtures with `params=`, and `pytest_generate_tests`/`metafunc.parametrize`.
- `references/markers-skip-xfail.md` (383 lines): builtin marker table, custom markers + registration (`markers` ini, `pytest_configure`), the strict family + the 9.0 `strict` umbrella, applying marks, `skip`/`skipif`, `xfail` (`condition`/`reason`/`raises`/`run`/`strict` + `--runxfail`), runtime `pytest.skip`/`xfail`/`importorskip`, and `-m` boolean expressions.
- `references/assertions.md` (476 lines): plain `assert` + rewriting (`register_assert_rewrite`, `--assert=plain`, `PYTEST_DONT_REWRITE`), `pytest.raises` (`match=`, `ExceptionInfo`, `check=` 8.4+, tuple, legacy callable, last-line gotcha), exception groups (`RaisesGroup`/`RaisesExc` 8.4+, `group_contains`), `pytest.warns`/`deprecated_call`, `pytest.approx` (rel/abs/`nan_ok`/Decimal/datetime + pitfalls), and the `fail`/`skip`/`xfail`/`exit`/`importorskip` imperatives.
- `references/cli-usage.md` (378 lines): invocation (`python -m pytest`, `--pyargs`, `@argfile` 8.2+, `pytest.main`), selection (node ids, `-k`, `-m`), `-x`/`--maxfail`, cache reruns (`--lf`/`--ff`/`--nf`/`--sw`/`--cache-show`), output/report flags (`-v`/`-q`, `-r`, `--capture`/`-s`, `--show-capture`), traceback styles (`--tb`, `-l`, `--full-trace`), `--durations`, collection (`--collect-only`/`--co`, `--ignore`, `--deselect`), debugging (`--pdb`/`--trace`/`--pdbcls`), plugin toggles (`-p`, `--disable-plugin-autoload` 8.4+), warnings (`-W`, `--max-warnings` 9.1+), `--junit-xml`, and exit codes **0–6** (code 6 = `MAX_WARNINGS_ERROR`, 9.1+).
- `references/configuration.md` (386 lines): config file formats + full precedence (incl. `pytest.toml`/`.pytest.toml` and native `[tool.pytest]`, both 9.0+, atop the four classic sources), the rootdir/configfile discovery algorithm, key ini options grouped (collection, `addopts`/`minversion`/`required_plugins`, markers + the 9.0+ `strict_*` family, warnings incl. `max_warnings` 9.1+, logging, tmp_path/output/misc), `--import-mode` (`prepend`/`append`/`importlib`) with src-layout & `__init__.py` implications, `pythonpath`/`consider_namespace_packages` (8.1+), and `conftest.py` layering.
- `references/plugins-hooks.md` (392 lines): the plugin model & three plugin kinds, using third-party plugins (xdist/cov/asyncio/mock/randomly/django/timeout), listing/controlling (`--trace-config`, `-p`/`-p no:`, `--disable-plugin-autoload` 8.4+), the startup discovery order, writing `conftest.py` plugins, installable `pytest11` entry-point plugins + assertion rewriting, the hook system (`hookspec`/`hookimpl`, `firstresult`, `wrapper=True` 8.0+ vs `hookwrapper=True`, `tryfirst`/`trylast`, `Stash`), a key-hooks reference table, and testing plugins with `pytester` (+ legacy `testdir`).
- `references/version-features.md` (254 lines): a consolidated `feature → minimum pytest version` lookup (pytest 7.0 → 9.1, ~70 source-cited rows each citing its `#issue`/`#pr`), a "what's bedrock (intentionally untagged)" list, a "commonly mis-pinned items" section, version-check guidance (`pytest --version`, `pytest.version_tuple`, `importlib.metadata.version`, `minversion`, `required_plugins`), and per-area release highlights.
- Inline `(pytest N.M+)` version annotations throughout, every pin verified against the in-repo changelog and consolidated in `references/version-features.md`. **Bedrock** (features stable since the pytest 6.x era or earlier) left **unannotated** ("unlisted = long-standing"); the taggable surface starts at **pytest 7.0**. Mirrors the postgres-sql/uv skills' structure, tone, and version-annotation approach.
- Documents verified, commonly-mis-pinned nuances: `RaisesGroup`/`RaisesExc`, `capteesys`, and `pytest.raises(check=)` are all **8.4** (not earlier); native TOML `[tool.pytest]` and `pytest.toml`/`.pytest.toml` are **9.0** while `[tool.pytest.ini_options]` is bedrock (6.0); the `strict` mode umbrella + `strict_xfail` alias are **9.0** while `xfail_strict` is bedrock (default `False`, never flipped) and the two are equivalent (neither deprecated); `--import-mode=importlib` exists since 6.0 but `prepend` is still the default through 9.1; exit codes now run **0–6** with code 6 (`MAX_WARNINGS_ERROR`) added in **9.1**.
